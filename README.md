# playbook-sync (pbs)

Sync AI agent playbooks (skills, rules, AGENTS.md) across multiple projects with bidirectional contribution support.

Designed for teams that maintain a shared knowledge base (e.g. [playbook-cocos](https://github.com/wenext-limited/playbook-cocos)) and need to distribute it to projects using different AI tools — **OpenCode**, **Cursor**, **GitHub Copilot**, **Claude Code**, and more.

## Features

- **Multi-source**: Git repos, local paths, git submodules
- **Multi-target**: Automatically formats and writes to each AI tool's expected location
  - OpenCode: `.opencode/skills/<name>/SKILL.md`
  - Cursor: `.cursor/rules/<name>.mdc` (with frontmatter conversion)
  - Copilot: `.github/copilot-instructions.md` (merged)
  - Claude Code: `.claude/skills/<name>/SKILL.md`
- **Auto-discovery**: Scans `skills/*/SKILL.md`, `rules/**/*.md`, `docs/**/*.md`, `AGENTS.md`, `README.md`
- **Lockfile**: Tracks synced content with checksums and git refs
- **Contribute back**: Detect local modifications and copy them back to the source repo — with optional auto-commit/push on a feature branch for PR workflow
- **Watch mode**: Real-time sync on source changes

## Install

```bash
npm install -g playbook-sync
```

Or run directly with npx:

```bash
npx playbook-sync <command>
```

## Quick Start

```bash
# 1. Initialize in your project
cd my-game-project
pbs init

# 2. Add a playbook source (git URL)
pbs add https://github.com/wenext-limited/playbook-cocos.git

# Or add a local path
pbs add --local ../playbook-cocos

# Or add a git submodule
pbs add --submodule vendor/playbook-cocos

# 3. Sync to all enabled AI tool directories
pbs sync

# 4. Check status
pbs status

# 5. Watch for source changes (auto-sync)
pbs watch
```

## Commands

### `pbs init`

Creates a `playbook-sync.yaml` config file in the current directory with default target settings.

### `pbs add <source>`

Add a playbook source to the config.

| Option | Description |
|--------|-------------|
| `-n, --name <name>` | Custom source name (default: inferred from URL/path) |
| `-r, --ref <ref>` | Git branch, tag, or commit (default: `main`) |
| `-l, --local` | Treat source as a local path |
| `-s, --submodule` | Treat source as a git submodule |
| `-i, --include <patterns...>` | Include only matching skill patterns |

### `pbs sync`

Resolve all sources, discover content, and write to enabled target directories. Updates the lockfile.

### `pbs status`

Show sync status for each source. Reports:
- Whether files are in sync with the lockfile
- Locally modified files (OpenCode targets)
- Whether the source has new commits upstream

### `pbs contribute`

Push local modifications back to the playbook source repository.

| Option | Description |
|--------|-------------|
| `-s, --source <name>` | Source name to contribute to (default: first source) |
| `-b, --branch <name>` | Branch name for the contribution |
| `-m, --message <msg>` | Commit message |
| `-p, --push` | Auto-push to remote (creates branch + commit + push) |
| `-d, --dry-run` | Show what would be contributed without applying |

**Typical contribution workflow:**

```bash
# 1. Edit a synced skill in your project
#    e.g., .opencode/skills/cocos-audio/SKILL.md

# 2. See what changed
pbs contribute --dry-run

# 3. Copy changes back to source (local only)
pbs contribute

# 4. Or auto-push to a feature branch for PR
pbs contribute --push --branch "fix/audio-docs" --message "fix: correct audio fade example"
```

### `pbs watch`

Watch source directories and auto-sync on changes. Only works with local and submodule sources.

## Configuration

`playbook-sync.yaml` example:

```yaml
version: 1
sources:
  - name: playbook-cocos
    type: git
    url: https://github.com/wenext-limited/playbook-cocos.git
    ref: main

  - name: local-playbook
    type: local
    path: ../playbook-cocos

targets:
  opencode:
    enabled: true
    skills_path: .opencode/skills
    agents_md: AGENTS.md
  cursor:
    enabled: true
    skills_path: .cursor/rules
    mdc_globs:
      - "**/*.ts"
      - "**/*.js"
  copilot:
    enabled: false
    skills_path: .github/copilot-instructions.md
    mode: merge
  claude:
    enabled: false
    skills_path: .claude/skills
    agents_md: AGENTS.md
```

## Lockfile

`playbook-sync.lock.yaml` is auto-generated by `pbs sync`. It records:

- The resolved git commit hash for each source
- The file list with SHA-256 checksums
- Sync timestamp

Commit the lockfile to your repo to ensure reproducible syncs across the team.

## How It Works

```
playbook-cocos (source repo)        Your Game Project
├── skills/                         ├── .opencode/skills/     ← OpenCode
│   ├── cocos-audio/SKILL.md  ───►  │   └── cocos-audio/SKILL.md
│   ├── oops-framework/SKILL.md     ├── .cursor/rules/        ← Cursor
│   └── ...                         │   ├── cocos-audio.mdc
├── rules/                          │   └── ...
│   └── common/coding-style.md      ├── playbook-sync.yaml
├── AGENTS.md                       ├── playbook-sync.lock.yaml
└── README.md                       └── AGENTS.md

        pbs sync ──►                        ◄── pbs contribute
```
